Be Careful With Floats

Floats pull affected elements out of the normal document flow and fix their display relative to the margin of their container and all content with static (default) or relative positioning will flow around it.

This paragraph tag picks up default positioning, which is static. It lives inside a div tag which has a relative positioning rule applied to it. It is flowing around the floated image, and so is the h2 tag above. For this reason, you have to be very conscious of the order of your markup and what exactly you've floated. If you do not expressly want the automatic flowing-around behavior for the non-floated content, don't create a float rule.

Floats: Ur Doin It Wron:

How to avoid wasting time trying to fix what will only get progressively worse until this.

  • You find yourself writing 3+ rules for no reason other than to control or compensate for the float.
  • You find yourself attempting to display multiple elements side-by-side by applying multiple floats.
    • ...especially if you're not floating them all to the same margin (ie, some are left, some are right)
    • ...double especially, if the elements are contiguous in the markup, and are not the columns of your multi-column layout
    • ...or if you're applying clears, absolute positioning, or margin offsets to nearby elements to fix their layout.
  • You find yourself adding empty markup with no content that serves no purpose beyond applying a clear rule.
  • You're trying to style nested navigation.

Tasks that where floats are often appropriate:

  • Arranging images within the main content text
  • Positioning sidebar-type containers, such as secondary navigation or ad blocks offset from the main content
  • Displaying an entire main navigation bar to the right or left
  • Magazine-style pullquotes, blockquotes, straplines, etc.
